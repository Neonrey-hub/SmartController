### ASR WebSocket 管理（AsrWebSocketManager）
- 职责
  - 连接建立、消息接收与解析、结果拼接与动态修正、断开清理
  - 将音频数据编码为 Base64 并发送
- 关键流程
  - 连接：生成 URL -> 创建 WebSocket -> 监听 open/message/error/close
  - 发送：起始帧 -> 音频帧 -> 结束帧
  - 结果：缓存乱序结果、动态修正、拼接文本、区分中间/最终结果
- 性能与稳定性
  - 使用数组缓存结果，按序列号拼接，减少 UI 抖动
  - 异常分支清晰，错误日志完整

```mermaid
flowchart TD
Start(["开始"]) --> Connect["连接 WebSocket"]
Connect --> SendStart["发送起始帧"]
SendStart --> Loop{"循环发送音频帧"}
Loop --> |有数据| SendAudio["编码并发送音频帧"]
SendAudio --> Loop
Loop --> |无数据| SendEnd["发送结束帧"]
SendEnd --> WaitResult["等待识别结果"]
WaitResult --> Parse["解析消息/缓存乱序结果"]
Parse --> Merge["拼接文本/动态修正"]
Merge --> IsFinal{"是否最终结果?"}
IsFinal --> |是| Close["断开连接"]
IsFinal --> |否| WaitResult
Close --> End(["结束"])
```

## 性能考虑
- WebSocket 连接与消息处理
  - 使用缓存数组按序列号拼接结果，避免频繁 UI 更新
  - 在收到最终结果后主动断开连接，释放资源
- 音频采集
  - 合理设置采样率与缓冲区大小，平衡延迟与质量
  - 在页面销毁或不再使用时及时 stop 与 release
- 权限检查
  - 避免重复请求权限，先检查再请求
- 日志与调试
  - 在关键路径打印日志，便于定位问题
  - 使用 try/catch 包裹异步操作，防止未捕获异常导致崩溃

## 故障排查指南
- WebSocket 连接失败
  - 检查鉴权 URL 生成是否正确，Host、Date、Authorization 是否完整
  - 查看 open/error/close 回调日志，定位具体错误码
- 音频无法采集
  - 确认麦克风权限已授予
  - 检查采样率、通道与编码配置是否匹配
- 识别结果异常
  - 观察动态修正（rpl）逻辑是否被触发
  - 确保 Base64 编码与 UTF-8 解码正确
- 返回确认无效
  - 确认 handleBackPress 返回值与定时器状态
  - 在切换 Tab 时调用 reset 清理状态

## 结论
本项目的工具类与实用程序围绕“统一常量、清晰职责、可扩展性”展开。日期时间工具、颜色与尺寸常量为界面一致性提供基础；权限、音频采集与 ASR 管理器构成完整的语音识别链路。建议后续在以下方向持续演进：
- 引入主题上下文与动态切换机制
- 扩展日期工具的国际化与时区支持
- 增加性能监控埋点与内存使用统计
- 完善错误分类与统一提示策略
